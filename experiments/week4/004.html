Week4 - Experiment4

AngularJS - Event Management Application II

This experiment improves the last experiment, by supporting rough seraching on the events, and sorting events according to different property.

Experiment Description:

  1. Ths app is on EventManager-II.html, which is shown in the following iframe.
  2. User can input some keyword, such as "NEU" in the search field, the controller would recognize it immediately and updates the table with only satisfied events in it.
  3. On the other hand, user can also choose which property on the event they want to sort by. For instance, user can sort the event by "Time".

Experiment:

Click button on top-left corner to view in new page

Code Analysis:

- HTML Snippet
<div class="well">
    <label for="filterInput">Search: </label>
    <input type="text" ng-model="query" id="filterInput" placeholder="">
    <label for="filterInput">Sort By: </label>
    <select ng-model="orderKey">
        <option value="eventName">Event Name</option>
        <option value="time">Time</option>
        <option value="place">Place</option>
    </select>
</div>
                    
<tr ng-repeat="e in events | filter: query | orderBy: orderKey">
    ...
</tr>
  1. Include angular.min.js and EventManager-II.js. (Only change in EventManager-II.js is the way it generate initial events)
  2. Add two more critiria filter: query and orderBy: orderKey in the directive ng-repeat. Which leads the controller to used query as the filter condition, and orderKey as the order condition.
  3. Bind <input> with element query, and bind <select> with element orderKey, by setting their ng-module directive. As mentioned above, the value of these two element would be used in ng-repeat of <tr> as the critira of filter and order events.

UI - Advanced Sign Page

This experiment focus on redesign the sign in page, and provide links to other user case, such as "Register" and "Forgot Password".

Experiment Description:

  1. As shown in the following iframe, the user may use the provided Register link, if they don't already have an account. (Register page not invented yet)
  2. If the user has an account, they can directly login by inputting their username and password. Or if their forgot their password, they can use Forgot Password? link to reset their password.

Experiment:

Click button on top-left corner to view in new page

Operation Analysis:

- HTML Snippet (signin.html) 

<div class="form-group">
    <div class="input-group">
        <span class="input-group-addon">
            <i class="glyphicon glyphicon-user"></i>
        </span>
        <input class="form-control" placeholder="Username" type="text"
            ng-model="user.username" autofocus required>
    </div>
</div>
...
<div class="row">
    <div class="col-xs-8">
        <b><a href="">Forgot password?</a></b>
    </div>
    <div class="col-xs-4 text-right">
        <button class="btn btn-success" type="submit"
        ng-click="signin(user)">Sign in</button>
    </div>
</div>
  1. User class "input-group" where a "input-group-addon" is supported to compose the input fields. These can provide user an intuitive look on the page. Also the input field has informative placeholder to remind user what to input.
  2. Notice that the two input field are mapped with ng-model, which will be used as parameters for the ng-click function signin(user)

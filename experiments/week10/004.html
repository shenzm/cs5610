UI - Profile Page I

This experiment build up the profile page of the user, which can be viewed by logged in users. It provide basic information of the related user, as well as the app-related info.

Experiment Description:

  1. Notice that we are trying to visit the profile page of the web application, but we will be redirected to signin page, if we are not previously logged in.
  2. After logged in we can see the profile page of the user. The profile page consist of two part, we will focus on the left-col in this experiment.
  3. Username: aliceW (asd@asd.com)
  4. Password: asd

Experiment:

Click button on top-left corner to view in new page

Operation Analysis:

- HTML Snippet (profile.html) 

<div class="col-sm-3 left-col">
    <div class="panel panel-default">
        <div class="header"></div>
        <div class="panel-body">
            <div class="text-center">
                <img class="img-responsive thumbnail" src="http://placehold.it/300x300">
            </div>
            <div class="profile-summary">
                <p>
                    FirstName: {{currentUser.firstName}}
                </p>
                <p>
                    LastName: {{currentUser.lastName}}
                </p>
                <p>
                    Email: {{currentUser.email}}
                </p>
            </div>
        </div>
    </div>
</div>
  1. The <div> consist of a header and a panel-body.
  2. The header is used as a UI component, which only adds a bold line at the upper border.
  3. The <img> tag with class thumbnail is used to display the picture of the user, and the <div> tag with class profile-summary are used to display the basic information of the user.

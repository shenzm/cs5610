Week10 - Experiment2

UI - Advanced navigation bar

This experiment focus on improve the existing navbar by adding the logic to only expose certain links to users according to their login state.

Experiment Description:

  1. As shown in the following iframe, if user is not logged in, then only the Sign In link shows on the navbar.
  2. Otherwise, if the user has logged in, then a dropdown link will show on the navbar, in which are two links: Profile and Sign Out
  3. Username: aliceW (asd@asd.com)
  4. Password: asd

Experiment:

Click button on top-left corner to view in new page

Operation Analysis:

- HTML Snippet (index.html) 

<ul class="nav navbar-nav navbar-right">
    <li ng-show="currentUser==null">
        <a href="#/signin">
            <span class="glyphicon glyphicon-log-in"></span> Sign In
        </a>
    </li>
    <li ng-show="currentUser!=null" class="dropdown">
        <a class="dropdown-toggle" data-toggle="dropdown" data-target="#">
            <span class="glyphicon glyphicon-user"></span> Welcome, {{currentUser.username}}
            <b class="caret"></b>
        </a>
        <ul class="dropdown-menu">
            <li>
                <a href="#/profile">
                    <span class="glyphicon glyphicon-info-sign"></span> Profile
                </a>
            </li>
            <li>
                <a ng-click="signout()">
                    <span class="glyphicon glyphicon-log-out"></span> Logout
                </a>
            </li>
        </ul>
    </li>
</ul>
  1. As shown, we use ng-show="currentUser==null" to make sure the link to "#/signin" only shows if the user are not logged in.
  2. Use ng-show="currentUser!=null" to make sure the link to "#/profile" and Logout only shows if the user has already logged in.
  3. Notice that the Profile page is protected by controller checking login state.

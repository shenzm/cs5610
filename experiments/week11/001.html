Week11 - Experiment1

Register Page I - UI

Since we already create a Log in page in previous experiments. This experiment focus on create a registration page for the website.

Experiment Description:

  1. As shown in the following iframe, user can jump to the register page by clicking the navigation link if they don't already have an account. Notice that they can also jump from the sign in page.
  2. The UI is really simple, it just contains three input field, which requires the user to input an Email address, a password and a password confirmation.

Experiment:

Click button on top-left corner to view in new page

Operation Analysis:

- HTML Snippet (register.html) 

<form>
    <div class="form-group">
        <div class="input-group">
            <span class="input-group-addon">
                <i class="glyphicon glyphicon-envelope"></i>
            </span>
            <input class="form-control" placeholder="Email Account" type="email"
                    ng-model="user.email" ng-change="clearWarning('email')" required>
        </div>
    </div>
    ...

    <hr>

    <div class="row">
        <div class="col-sm-8">
            <input ng-model="confirm" type="checkbox" required>
            <label>I've read the <a href="#/terms_conds">Terms and Conditions</a></label>
        </div>
        <div class="col-sm-4 text-right">
            <button class="btn btn-success" type="submit"
                    ng-click="register(user, confirm)">Register</button>
        </div>
    </div>

    <hr ng-show="warnings != null">

    <div class="alert alert-warning" role="alert" ng-show="warnings != null">
        <p ng-repeat="w in warnings">{{w}}</p>
    </div>

</form>
  1. The basic register form only consist of four input field: Email address, password and password confirmation. Notice that user have to click on the checkbox to confirm that they have read the terms and conditions
  2. There is also a alert <div> at the bottom of the page, which might shows after user submit an invalid registration form.

On the first week, I focused on basic web page technologies such as HTML5, CSS. During which I tried how to do responsive web page designs. Following are the five experiments I have done:
1) HTML5 - New Semantic Elements
2) HTML5 - New Form Elements
3) CSS - Unified Page Style
4) CSS - CodeBox Utility Class
5) CSS - Responsive Web Page

HTML5 - New Semantic Elements

Try out new semantic elements of HTML5 which are not supported in previous HTML versions. Such as <nav>, <details>, <figcaption>. These new semantic tags are supported to provide more meaningful keyword for web searching.

HTML5 - New Form Elements

Try out new form elements of HTML5 which are not supported in previous HTML versions. Such as <datalist>, <output>. <datalist> tag is similar to <select> tag, but it allows user to input new options as well as choosing from exising options. <output> tag can be used to represents the result of an input.

CSS - Unified Page Style

I defined a general style of experiment page using pure CSS in this experiment. So it can be used across all the experiments. Basically, it consist of an banner that shows the experiment number and week number, a navigation bar which lead the users to the useful links at the bottom of the page. And for the experiment content is consist of sub-items under the experiment and the related links.

CSS - CodeBox Utility Class

This is an utiliy css class is defined to support display well-indented code snippet inside of the webpage. It used a border and different background to differentiate the code snippet with the other paragraphs in the web page.

CSS - Responsive Web Page

In this experiments, I applied the responsive design concept on the web page, which will dynamically change the font-size of the html content, layout patterns and the backgound color according to the width of the web browser.
